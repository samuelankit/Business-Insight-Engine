# GoRigo — AI Business Operating System

## Overview

GoRigo (gorigo.ai) is a voice-controlled AI business operating system mobile app designed for UK business users. It enables multi-business management, offers an AI agent dashboard, integrates Telnyx for communications, and ensures UK GDPR/PECR compliance. A core feature is the "Bring Your Own Key" (BYOK) model for Large Language Models (LLMs) like OpenAI and Anthropic, allowing users to provide and manage their own API keys. The project aims to provide a comprehensive, AI-powered platform to streamline business operations through intelligent automation and communication tools.

## System Architecture

The project is built as a monorepo using `pnpm workspaces` with a Node.js 24 backend and an Expo mobile frontend. TypeScript 5.9 is used throughout.

**Backend (artifacts/api-server):**
- **API Framework:** Express 5.
- **Database:** PostgreSQL with Drizzle ORM for schema definition and interaction.
- **Validation:** Zod for data validation, integrated with `drizzle-zod`.
- **Security:** AES-256-GCM encryption for sensitive data like API keys, requiring an `ENCRYPTION_KEY` at startup. Rate limiting enforced via `express-rate-limit`.
- **Compliance:** GDPR-compliant user and business data purging mechanisms.
- **AI Orchestration:** Supports OpenAI `tools` and Anthropic `tool_use` for function calling. Conversation history is managed by token count.
- **Knowledge Base (RAG):** Utilizes `pgvector` for embeddings (OpenAI `text-embedding-3-small`) of various document types. Chunks are stored in `knowledge_chunks` with HNSW indexing, enabling context injection into LLM prompts.
- **Tooling System (BYOA):** A modular system allows dynamic registration of tools. Tools define functions with JSON Schema parameters and `execute` methods. Credentials are encrypted and decrypted on-the-fly. Dynamic system prompts enhance LLM awareness of available tools.
- **Agent Scheduling:** A background scheduler manages agent execution based on configurable schedules (hourly, daily, weekly, custom). Significant actions from scheduled agents can be queued for user approval.
- **Error Handling:** Global Express error middleware for sanitized responses.
- **SSE:** Server-Sent Events are used to stream real-time status updates, including tool execution progress.

- **BYOK**: Users provide their own OpenAI/Anthropic/ElevenLabs API keys; stored AES-256-GCM encrypted in DB
- **Premium Voice Engine**: Platform TTS uses Google Cloud TTS Neural2 (en-GB-Neural2-B) via `GOOGLE_TTS_API_KEY`. Optional BYOK ElevenLabs premium upgrade. POST /tts returns audio/mpeg. Voice preferences (provider, voiceName, speechRate) stored in `voice_preferences` table, managed via GET/PUT /voice-preferences.
- **Realtime AI Calls**: After Telnyx disclosure plays, inbound/outbound calls upgraded to live AI conversation via OpenAI Realtime API WebSocket bridge (`realtimeCall.ts`). Requires user's OpenAI key. Transcripts captured, call minutes billed via wallet (15p per realtime_call).
- **Encryption key**: `ENCRYPTION_KEY` env var (64-char hex, 32 bytes); required at startup — server will refuse to start without it
- **Rate limiting**: express-rate-limit — 100 req/min general, 20 req/min AI endpoints, 10 req/min auth, 200 req/min webhooks
- **Input validation**: Zod middleware on critical POST routes (orchestrate, businesses, keys, tools/connections)
- **GDPR**: `purgeUserData(userId)` and `purgeBusinessData(businessId)` cascade-delete all associated records; DELETE /account endpoint
- **AI Disclosure (UK Compliance)**: Every outbound/inbound call plays a TTS disclosure message before connecting. Uses Telnyx Call Control `speak` action. Configurable per business via `disclosure_message` column in `telnyx_configs`. Default: "This call is being made with AI assistance on behalf of {businessName}. You may request to speak with a human at any time." Answering machine detection auto-hangs-up on voicemail. Call states: ringing → answered → disclosure_played → ai_active → completed (or voicemail_detected, failed). `disclosure_compliant` boolean on communications table tracks compliance.
- **Gemini Live AI Call Engine**: Fully automated, platform-managed real-time AI phone calls. After disclosure plays, Telnyx media streaming starts and audio is bridged bidirectionally to Google Gemini Live API (`gemini-2.0-flash-live`). Audio codec: Telnyx µ-law 8kHz ↔ PCM 16kHz (Gemini input) / 24kHz (Gemini output). Transcripts accumulated from Gemini's input/output transcriptions. On hangup: transcript saved, summary generated via Gemini, per-minute billing charged to wallet (`ai_call` at 15p/min). Platform Gemini usage tracked separately (`ai_call_platform`). `GEMINI_API_KEY` env var required (server-side only, never exposed to users). WebSocket server on `/api/telnyx-stream` for Telnyx media streams. Key files: `src/lib/geminiLive.ts`, `src/lib/callSession.ts`, `src/lib/audioCodec.ts`, `src/ws/telnyxStream.ts`.
- **Context window management**: Conversation history truncated by approx token count before sending to LLMs
- **Voice-First Platform Tools**: 7 platform tools registered as "credentialType: platform" builtins that execute directly against the DB without external credentials. They receive `{userId, businessId}` as JSON context. Always available (no connection needed). Tools: businesses, agent_management, team, settings, wallet, referrals, comms.
- **Platform Tool SSE Events**: Platform tools can return `_event` and `_eventData` fields which get forwarded to the mobile client as SSE events. Mobile handles: `settings_update` (theme changes), `business_switched/created/deleted` (refetch businesses), `checkout_url` (open payment link).
- **Tool timeouts**: All tool execute() calls wrapped with 30s timeout
- **SSE abort handling**: Client disconnect aborts LLM calls and tool execution via AbortController
- **Global error handler**: Express error middleware sanitises error responses in production
- **Database indexes**: Composite indexes on agent_logs, usage_events, conversations, communications, tool_connections, agents, businesses
- **Foreign key constraints**: businesses→users, agents→users/businesses, agent_logs→agents, conversations→users
- **Business management**: First business is auto-set as active; only one active at a time
- **Admin console**: SuperAdmin access via `ADMIN_DEVICE_ID` env var matching a user's device_id. Protected by `adminAuth` middleware. Mobile screens at `/admin/*`. Admin voice tools registered as `admin_platform` builtin tool. Users can be suspended via `suspended` boolean on users table.
- **Onboarding**: AsyncStorage keys `@gorigo/onboarding_complete` and `@gorigo/active_business_id`
- **API base URL**: Set via `setBaseUrl()` in `_layout.tsx` using `EXPO_PUBLIC_DOMAIN`
- **Wake Word / Hands-Free Mode**: `WakeWordProvider` context wraps the app in `_layout.tsx`. Uses audio metering for voice activity detection to trigger recording. Preference persisted in AsyncStorage (`@gorigo/wake_word_enabled`). Toggle in Settings > Voice. VAD (Voice Activity Detection) auto-stops recording after ~1.5s silence. Pauses wake word listening during recording/TTS, resumes after.

**Frontend (artifacts/mobile):**
- **Framework:** Expo (SDK 54) with `expo-router v6` for navigation.
- **UI/UX:** Supports Light/Dark/System themes with a Gold accent, `Inter` font. Theme persistence is handled via AsyncStorage. Styling uses a `createStyles` factory pattern.
- **State Management:** React Query hooks for API interactions generated by Orval.
- **Offline Capabilities:** Features `useNetworkStatus`, `useOfflineQueue` (with AsyncStorage for persistent command queuing and sequential sync), and `useDataCache` (24h TTL) for robust offline operation.
- **Voice Interaction:** Includes a `WakeWordProvider` for hands-free voice control and voice activity detection (VAD).
- **Notifications:** Push notification registration and deep-linking support.

**Data Flow & Structure:**
- **Monorepo Structure:** Divided into `artifacts/` (api-server, mobile), `lib/` (api-spec, api-client-react, api-zod, db), and `scripts/`.

## Packages

## Tool Connection System (BYOA)

- **Architecture**: Modular tool registry at `artifacts/api-server/src/lib/tools/`
- **Adding a new tool**: Create a `ToolDefinition` object and call `registerTool()` at server startup
- **Tool interface**: Each tool declares name, description, credentialType, functions (with JSON Schema params + execute method)
- **Function calling**: Orchestrate endpoint supports OpenAI `tools` param and Anthropic `tool_use`. Tool calls loop up to 5 rounds.
- **Credentials**: Stored encrypted (AES-256-GCM) in `tool_connections` table; decrypted on-the-fly during tool execution
- **Dynamic system prompt**: When a user has connected tools, the system prompt is enhanced to list available tool functions
- **SSE status updates**: During tool execution, `tool_status` events are streamed to the mobile app (e.g. "Using gmail...")

### `artifacts/api-server` (`@workspace/api-server`)

Express 5 API server. Routes in `src/routes/`. Crypto utility for AES-256-GCM at `src/lib/crypto.ts`. Tool registry at `src/lib/tools/`.

**Google Integration**: OAuth 2.0 flow at `src/routes/google.ts`, tool execution logic at `src/lib/googleTools.ts`. Requires `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI` env vars. The orchestrate route (`src/routes/orchestrate.ts`) supports function calling with Google tools when connected. Dependencies: `googleapis`.

**Notion Integration**: OAuth 2.0 flow at `src/routes/notion.ts`, auth helpers at `src/lib/notionAuth.ts`, tool definition at `src/lib/tools/builtins/notion.ts`. Requires `NOTION_CLIENT_ID`, `NOTION_CLIENT_SECRET`, `NOTION_REDIRECT_URI` env vars. Functions: search_pages, create_page, read_page, list_databases, query_database, add_database_entry. Credentials stored in `tool_connections` table.

**Trello Integration**: API key + token tool at `src/lib/tools/builtins/trello.ts`. No OAuth required — users provide API key and token as JSON `{"key": "...", "token": "..."}`. Functions: list_boards, list_board_lists, list_cards, create_card, move_card, search_cards. Uses Trello REST API v1.

**Stripe Integration**: Registered as a builtin tool (`src/lib/tools/builtins/stripe.ts`). Functions: get_balance, list_payments, list_invoices, create_invoice, get_customer, list_customers. Uses Stripe REST API with user's secret key (api_key credential type). Currency defaults to GBP.

**Stripe Checkout (Payment Processing)**: Subscription billing via Stripe Checkout at `src/lib/stripe.ts` and `src/lib/usage.ts`. Requires `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRO_PRICE_ID`, `STRIPE_UNLIMITED_PRICE_ID` env vars. POST /usage/upgrade creates a Stripe Checkout session for paid plans (Pro £19/mo, Unlimited £49/mo) and returns checkoutUrl. POST /webhooks/stripe handles checkout.session.completed, customer.subscription.updated, customer.subscription.deleted events. GET /usage/billing-portal returns Stripe Customer Portal URL. DB columns: user_subscriptions.stripe_customer_id, user_subscriptions.stripe_subscription_id.

**Xero Integration**: OAuth 2.0 flow at `src/routes/xero.ts`, auth logic at `src/lib/xeroAuth.ts`, tool at `src/lib/tools/builtins/xero.ts`. Requires `XERO_CLIENT_ID`, `XERO_CLIENT_SECRET`, `XERO_REDIRECT_URI` env vars. Functions: get_organisation, list_invoices, create_invoice, list_expenses, get_vat_return, list_bank_transactions. Stores OAuth tokens as encrypted JSON in tool_connections table. Automatic token refresh before execution.

**Slack Integration**: OAuth 2.0 flow at `src/routes/slack.ts`, auth logic at `src/lib/slackAuth.ts`. Tool definition at `src/lib/tools/builtins/slack.ts` with functions: list_channels, read_channel_messages, send_message, set_reminder, search_messages. Requires `SLACK_CLIENT_ID`, `SLACK_CLIENT_SECRET`, `SLACK_REDIRECT_URI` env vars. Scopes: channels:read, channels:history, groups:read, groups:history, chat:write, reminders:write, users:read, search:read, im:write.

**Agent Scheduling Engine**: Background scheduler at `src/lib/scheduler.ts`. Checks every 60s for agents with `next_run_at` in the past and `is_active = true`. Runs agents with full tool access (same as orchestrate endpoint). Significant actions (send_email, create_invoice, etc.) are queued as pending approvals in `agent_pending_actions` table instead of executing immediately. Approvals expire after 24h. Max 3 concurrent scheduled runs. Schedule types: none, hourly, daily (at specific time), weekly (day + time), custom (interval in minutes). Mobile UI: Schedule picker on agent detail screen, approval cards with approve/reject buttons, schedule status badges on agent list.

## Production Review Fixes Applied

### P0 Fixes (Critical)
- **Campaign wallet integration**: Campaign executor now calls `debitWallet()` for every SMS/call sent, with `checkWalletBalance()` pre-check. Campaign auto-pauses if wallet is depleted mid-batch.
- **Realtime call mid-call billing**: OpenAI Realtime path now has a 60-second billing interval (matching Gemini path). Checks balance each minute and terminates call if wallet empty. End-of-session billing only covers remaining unbilled minutes.
- **Billing portal device ID security**: Device IDs no longer passed in Stripe redirect URLs. Stored in sessionStorage instead. URL is cleaned on initial load.

### P1 Fixes
- **Low-balance notifications**: `debitWallet()` now triggers push notifications when balance drops below £5 (warning) or £1 (critical).
- **TTS interruption on wake word**: `stopSpeaking()` is called before recording starts in both the wake word handler and manual mic press.
- **Campaign completion notification**: Push notification sent when a campaign finishes all messages.
- **AI call max duration**: Both Gemini and OpenAI Realtime sessions enforce a 30-minute maximum call duration.
- **Orphan session sweeper**: Background interval (every 5 minutes) cleans up any call sessions older than 35 minutes that weren't properly terminated.

### P2 Fixes
- **E.164 phone validation**: Single contact creation validates phone number format (E.164: `+447700900000`). Bulk import silently skips invalid phone numbers and reports them as `skipped`.
- **STOP/unsubscribe handling**: Inbound SMS with STOP/QUIT/UNSUBSCRIBE/CANCEL/END keywords automatically marks the contact as DNC-listed and sends a confirmation reply. No further messages will be sent to DNC contacts.
- **PECR consent timestamps**: Both `contacts` and `telnyx_configs` tables now record `consent_at` timestamp when consent is granted. Consent timestamps are set on creation (if consent=true) and on toggle (only when changing from false→true).
- **Team join notifications**: Business owner receives a push notification when a new team member joins via invite code.
- **Knowledge processing notifications**: Users get a push notification when document processing completes (success or failure).
- **Contact phone index**: Added composite index on `(phone, business_id)` for efficient DNC lookups by inbound SMS.
